# AI Session Log — 2026-04-02

## User request summary
- Build a responsive grid for the portfolio research page that scales cleanly from 320px to 1440px
- Must satisfy WCAG AA contrast on body text and link states
- The grid should be drop-in compatible with our existing Tailwind 4 + custom CSS variables system

## Prompt used (paraphrased, not verbatim)
> "Convert this responsive grid from `min-width: 768px` (2 columns) to a 3-column layout at `min-width: 1024px`, preserving 24px gaps. All focus states must remain visible against both light and dark theme backgrounds. Use CSS variables for colour so theme switching keeps working. Do not introduce new dependencies."

## What the AI returned
- A `@media` query block with `grid-template-columns: repeat(3, minmax(0, 1fr))` at `min-width: 1024px`
- A `:focus-visible` rule using `outline: 2px solid var(--accent)` to stay theme-aware
- A small `gap` adjustment that scales 16px → 24px at the wider breakpoint

## Changes we accepted vs. rejected
- **Accepted:** the breakpoint structure, `minmax(0, 1fr)` for safe column shrinking, `:focus-visible` over `:focus`
- **Rejected:** the AI suggested adding a `prefers-reduced-motion` block that hard-disabled all transitions — we kept transitions but reduced their duration to 80ms so animation-sensitive users still get visual feedback
- **Edited:** swapped the AI's hard-coded gap values for our existing `--space-md` token so the grid matches the rest of the design system

## Verification performed
- Resized the viewport from 320px to 1440px in Chrome DevTools and confirmed no horizontal scroll at any width
- Ran axe-core (DevTools extension) on the research page — zero violations
- Checked colour contrast in DevTools: body text 8.2:1 (light), 11.4:1 (dark); link text 6.1:1 (light), 7.8:1 (dark) — all pass WCAG AAA, well above the AA target
- Tab-order audit: every card is reachable via keyboard with a visible focus ring on both themes
- Keyboard-only smoke test on the live deploy (GitHub Pages) confirmed no regressions

## Notes for the team
- All future grids should use `minmax(0, 1fr)` rather than `1fr` — prevents content overflow on long titles
- `:focus-visible` is preferred over `:focus`; otherwise mouse clicks leave a permanent ring on cards
- When AI suggests a `prefers-reduced-motion` block, sanity-check it: hard-disabling all motion can hide important state changes
