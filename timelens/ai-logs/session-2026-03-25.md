# AI Session Log — 2026-03-25

## User request summary
- Scaffold the 3D pocket-watch scene used as the first NFC reveal moment
- Component must work inside React 19 + Vite, and play well with our Tailwind layout
- Output should be deterministic enough that we can re-run it from prompt for any new keepsake (watch, photo frame, recipe notebook)

## Prompt used (paraphrased, not verbatim)
> "Generate a React Three Fiber component named `KeepsakeScene` that renders a low-poly pocket-watch model, supports `OrbitControls` from `@react-three/drei`, exposes a `idleRotation` prop to play a soft Y-axis rotation when true, and accepts a `meshUrl` prop so we can swap to other keepsakes (recipe book, photo frame). The component must mount inside an existing `<Canvas>` and not introduce any global side effects."

## What the AI returned
- A `KeepsakeScene.jsx` skeleton using `useFrame` for the rotation tick
- `<Suspense fallback={null}>` wrapping a `useGLTF`-loaded mesh
- An `OrbitControls` instance with `enableZoom={false}` and `enablePan={false}` to keep the interaction simple for elder users

## Changes we accepted vs. rejected
- **Accepted:** the `useFrame` rotation pattern, `Suspense` wrapping, prop shape (`idleRotation`, `meshUrl`)
- **Rejected:** the AI suggested fetching meshes from a public CDN (`https://threejs.org/examples/...`) — replaced with a local asset path under `public/assets/keepsakes/` so the prototype works offline and avoids licensing risk
- **Edited:** renamed `props.url` → `props.meshUrl` to match our naming convention; added a `data-testid` for our manual UI test pass

## Verification performed
- Ran `npm run dev` and visually confirmed the watch model renders and rotates idle
- Frame-rate sanity check: ~60fps on M-series Mac, ~45fps on Pixel 6 (Android Chrome)
- Manually tested the `OrbitControls` drag on touch devices to confirm pinch-zoom is correctly disabled
- Verified that swapping `meshUrl` to the recipe-book glTF works without code changes (proves the prop API is stable)

## Notes for the team
- All future keepsake meshes should live in `public/assets/keepsakes/` and be referenced relatively
- Do not commit AI-suggested external asset URLs — we hit a CORS issue on the first attempt before swapping to local paths
